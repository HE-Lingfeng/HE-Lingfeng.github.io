# HE-Lingfeng.github.io

Lingfeng He's personal website, published with GitHub Pages.

## Site direction

This site is designed as a calm, long-term academic homepage rather than a short-lived portfolio splash page. The structure should stay simple:

- `index.html`: homepage, selected work, research direction, contact.
- `css/style.css`: global visual system and compatibility styles for generated pages.
- `/posts/`: future writing index for notes, project logs, paper reading, and experiments.

## Maintenance plan

1. Keep the homepage curated. Add only selected publications, important projects, and current research directions.
2. Use posts for accumulation. Put paper notes, experiment write-ups, and project retrospectives under `/posts/` instead of expanding the homepage indefinitely.
3. Add one project page per mature project. Each project page should include motivation, method, links, citation, and representative figures.
4. Refresh the homepage quarterly. Update status, publications, links, and research focus without changing the whole design language.
5. Keep the stack lightweight. Prefer semantic HTML and CSS unless the site genuinely needs search, filtering, or richer interaction.
